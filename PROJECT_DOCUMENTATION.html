Project Overview

This is a minimal Django starter project designed for quick development and deployment. It includes a responsive portfolio site with a smooth carousel, contact form with feedback collection, and production-ready deployment configurations for multiple platforms (Railway, Render, PythonAnywhere).

Key Features: - Responsive design with mobile-first approach - Smooth carousel animation with keyboard & touch support - Contact form with feedback rating system - Production-ready deployment configurations - Admin interface for managing feedback - Static file optimization with WhiteNoise - Support for custom domain deployment

Local Development Setup

Prerequisites

  • Python: 3.10 or higher
  • Git: For version control
  • Terminal: bash/zsh/PowerShell

Step-by-Step Setup

Step 1: Clone the repository

cd ~/CodeWorld/AI-Projects
git clone https://github.com/ayushsatvara1012/django-portfolio.git
cd django-site

Step 2: Create virtual environment

python3 -m venv .venv

Step 3: Activate virtual environment

macOS/Linux:

source .venv/bin/activate

Windows:

.venv\Scripts\activate

Step 4: Upgrade pip and install dependencies

.venv/bin/python -m pip install --upgrade pip setuptools wheel
.venv/bin/python -m pip install -r requirements.txt

Step 5: Run database migrations

.venv/bin/python manage.py migrate

Step 6: Create superuser account

.venv/bin/python manage.py createsuperuser
# Follow prompts to create admin account

Step 7: Run development server

.venv/bin/python manage.py runserver 8000

Access the site: - Homepage: http://127.0.0.1:8000 - Admin panel: http://127.0.0.1:8000/admin

Key Files & Components

1. models.py - Database Schema

Defines the Feedback model for collecting user feedback:

class Feedback(models.Model):
    name = models.CharField(max_length=100)          # User's name
    email = models.EmailField()                       # Contact email
    message = models.TextField()                      # Feedback message
    rating = models.IntegerField()                    # Star rating (1-5)
    created_at = models.DateTimeField(auto_now_add=True)  # Timestamp

    def __str__(self):
        return f"Feedback from {self.name} ({self.rating} stars)"

Purpose: Stores user feedback with rating, automatically timestamped.

2. forms.py - Form Validation

Defines FeedbackForm for HTML form rendering and validation:

class FeedbackForm(forms.Form):
    name = forms.CharField(max_length=100, widget=forms.TextInput(
        attrs={'placeholder': 'Your name'}))
    
    email = forms.EmailField(widget=forms.EmailInput(
        attrs={'placeholder': 'your@email.com'}))
    
    message = forms.CharField(widget=forms.Textarea(
        attrs={'placeholder': 'Your message...', 'rows': 5}))
    
    rating = forms.ChoiceField(
        choices=[
            (1, '⭐ Poor'),
            (2, '⭐⭐ Fair'),
            (3, '⭐⭐⭐ Good'),
            (4, '⭐⭐⭐⭐ Very Good'),
            (5, '⭐⭐⭐⭐⭐ Excellent'),
        ],
        widget=forms.RadioSelect()
    )

Purpose: Validates user input before saving to database; renders form HTML with placeholders and emoji ratings.

3. views.py - Request Handlers

Implements 4 view functions:

def home(request):
    """Renders homepage with carousel and hero section"""
    return render(request, 'core/home.html')

def about(request):
    """Renders about page"""
    return render(request, 'core/about.html')

def contact(request):
    """Renders contact information page"""
    return render(request, 'core/contact.html')

def feedback(request):
    """Handles feedback form submission (GET/POST)"""
    if request.method == 'POST':
        form = FeedbackForm(request.POST)
        if form.is_valid():
            Feedback.objects.create(
                name=form.cleaned_data['name'],
                email=form.cleaned_data['email'],
                message=form.cleaned_data['message'],
                rating=form.cleaned_data['rating']
            )
            messages.success(request, f'Thank you {form.cleaned_data["name"]} for your feedback!')
            return redirect('home')
    else:
        form = FeedbackForm()
    return render(request, 'core/feedback.html', {'form': form})

Purpose: Routes HTTP requests to templates; handles form processing with validation and success messages.

4. urls.py - URL Routing

Defines application routes:

from django.urls import path

Deployment Guide

Railway (Recommended)

Advantages: - Easiest setup - Free tier available - Auto-deploys from GitHub - Built-in domain support

Steps:

  1. Push to GitHub:

    git add .
git commit -m "Prepare for deployment"
git push origin main

  2. Deploy to Railway:

    • Go to https://railway.app/
    • Sign up with GitHub
    • Click “New Project” → “Deploy from GitHub repo”
    • Select django-portfolio repository
    • Railway auto-detects Django and deploys

  3. Set Environment Variables:

    • In Railway dashboard → Variables tab, add:
    DJANGO_SECRET_KEY=your-secret-key-here
DEBUG=False
ALLOWED_HOSTS=.railway.app,.yourdomain.com

  4. Connect Custom Domain:

    • In Railway → Settings → Domains → Custom Domain
    • Add your domain (e.g., yourdomain.com)
    • Follow DNS setup instructions

Render

Advantages: - Similar to Railway - Reliable uptime - Free tier available

Steps:

  1. Create web service:

    • Go to https://render.com/
    • Click “New +” → “Web Service”
    • Connect GitHub repo

  2. Configure:

    • Build Command: pip install -r requirements.txt && python manage.py collectstatic --noinput
    • Start Command: gunicorn mysite.wsgi:application

  3. Set environment variables:

    DJANGO_SECRET_KEY=your-secret-key
DEBUG=False

Google Cloud Run

Advantages: - Pay-per-use pricing - Automatic scaling - Good for variable traffic

Steps:

  1. Enable Cloud Run API in Google Cloud Console
  2. Deploy using Cloud Build: gcloud run deploy
  3. Configure environment variables
  4. Connect domain

PythonAnywhere

Advantages: - Beginner-friendly - Python-specific platform - Free tier available

Limitations: - Free tier doesn’t support custom domains - Requires paid plan ($5/month) for custom domains

Steps:

  1. Sign up at https://www.pythonanywhere.com/
  2. Upload code via Git or zip
  3. Configure virtual environment
  4. Set up web app with WSGI file
  5. Configure domain (paid only)

Generate Secret Key

python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'

Troubleshooting

Issue Cause Solution
Static files not loading (404) collectstatic not run or wrong STATIC_ROOT Run python manage.py collectstatic --clear --noinput
Carousel glitchy/jumping CSS scroll-snap conflict with JS animation Ensure scroll-behavior: auto in CSS; JS controls smoothing
Feedback form not saving Database not migrated Run python manage.py migrate
Admin panel blank Missing static files Run python manage.py collectstatic
Port 8000 already in use Another process using port Use python manage.py runserver 8001 (different port)
Secret key error on deploy Environment variable not set Add DJANGO_SECRET_KEY to deployment platform settings
CSRF token missing/invalid Domain not in CSRF_TRUSTED_ORIGINS Add domain to settings.py or use environment config

Recent Improvements

Carousel Animation Fixes (Dec 2, 2025)

Problem: Carousel slides were glitchy, with jitter and jumping

Solutions Implemented:

  1. RequestAnimationFrame Animator
    • Replaced native scrollTo() with RAF-based animator
    • Uses easeInOutCubic easing function
    • Smooth 420ms animation duration
    • Cancellable animation IDs prevent overlapping
  2. Integer Alignment
    • All gaps rounded to integers: Math.round(gap)
    • Slide widths floored to integers: Math.floor(available / v)
    • Eliminates sub-pixel rendering jitter
  3. CSS Optimization
    • Set scroll-behavior: auto to let JS handle smoothing
    • Removed scroll-snap-type to eliminate browser conflicts
    • Reduced transition duration from 220ms to 160ms
  4. Touch Device Support
    • Added @media (hover: none) to disable hover effects on touch
    • Prevents layout shifts on tap
    • Improves touch interaction performance
  5. Animation State Guards
    • Check if (isAnimating && smooth) return; before starting animation
    • Prevents race conditions from rapid button clicks
